# EZ-Shell (EZS) 言語仕様書 v4.6

本仕様書は `index.html` に実装されている EZ-Shell (EZS) ランタイムの実挙動を基準に、抜け漏れがない形で定義します。  
実装と仕様が不一致な場合は、`index.html` の挙動を正とします。

---

## 1. 目的と対象
- **目的**: 教育・デモ用途の軽量スクリプト言語として、最小限の文法と見える実行結果を提供します。
- **対象環境**: ブラウザ内のランタイム（`index.html`）。
- **設計思想**: 「行単位コマンド + 最低限の制御構文 + 視覚出力」。

---

## 2. 字句と構文の共通ルール

### 2.1 行の扱い
- 入力は **行単位** で処理されます。
- 各行は **前後の空白がトリム** されます。
- **空行は無視** されます。
- コメントは **行頭が `//` の行のみ無視** されます。  
  行末コメント（例: `echo hi // comment`）はサポートされません。

### 2.2 コマンド形式
```
命令名 [引数1] [引数2] ...
```
- 命令名は **小文字推奨**（実装上は大小区別があり、正確な名前が必要）。
- 引数区切りは **1つ以上の空白**。

### 2.3 文字列
- **二重引用符 `"`** で囲むと文字列として扱われます。
- エスケープは **非対応**（`\"` 等は無効）。
- `"` を含む文字列は定義できません。

---

## 3. データと評価ルール

### 3.1 値の型
EZS は **動的型** で、値は以下のどちらかで保持されます。
- 数値（JavaScript の `Number`）
- 文字列（`String`）

### 3.2 変数参照
- 変数参照は `$name` 形式。
- 未定義変数は **数値評価では `0`** とみなされます。
- 条件式評価では **未定義変数は空文字 `""`** とみなされます。

### 3.3 評価の優先ルール（`getValue` 相当）
値の評価は以下の順で実行されます。
1. **`calc(...)` が含まれる場合**  
   - `calc(式)` の `式` を JavaScript の `eval()` で評価します。  
   - `calc()` は文字列中に含まれていれば評価対象になります。  
   - 正規表現が貪欲なため、複数の `calc()` を1行に書くと不正な結果になる可能性があります。
2. **`+` が含まれる場合（文字列連結）**  
   - `+` で分割し、各要素をトリム → 先頭末尾の `"` を除去 → 連結します。  
   - 演算の優先順位はありません（単純分割）。  
   - `+` 文字自体を含む文字列は直接表現できません。
3. **二重引用符で囲まれている場合**  
   - `"` を除去して文字列として返します。
4. **数値判定**  
   - `isNaN` 判定で数値なら `Number` へ変換します。
5. **その他**  
   - そのまま文字列として扱います。

## 4. 命令一覧

### 4.1 `echo`
```
echo [内容]
```
- 1行出力。  
- 出力色は **現在のテキスト色**（`color` / `reset` の状態）を使用。

### 4.2 `set`
```
set [変数名] [値]
```
- 変数へ代入。
- `[値]` は `getValue` ルールで評価されます。

### 4.3 `calc`
```
calc(式)
```
- `set` や `echo` などの値中に埋め込んで使用。
- `式` は JavaScript の `eval()` によって評価されます。  
  **安全ではないため、外部入力は入れないこと。**

### 4.4 `if`
```
if [条件式] { ... }
```
- 条件が `true` の場合のみブロックを実行。
- `{` は **同じ行に必要**。  
  例: `if $i > 0 {`
- `}` は **単独行** で閉じること。
- 条件式は JavaScript と同様に評価されます。  
  例: `==`, `!=`, `>`, `<`, `>=`, `<=`

### 4.5 `loop`
```
loop [条件式] { ... }
```
- 条件が `true` の間ブロックを反復。
- `if` と同じブロック規則。

### 4.6 `color`
```
color [色名 or CSS色コード]
```
- 出力色を変更します（例: `red`, `#00FF00`）。
- 以降の `echo` 出力に適用されます。

### 4.7 `reset`
```
reset
```
- 出力色をデフォルト（`#39ff14`）に戻します。

### 4.8 `clear`
```
clear
```
- ターミナル出力を全消去。

### 4.9 `wait`
```
wait [ミリ秒]
```
- 指定時間、処理を停止します。

### 4.10 `input`
```
input [変数名] [メッセージ]
```
- 入力待機を行い、Enter で確定。
- `[メッセージ]` がある場合は事前に出力します。
- 入力された文字列は **そのまま文字列として** 変数に格納されます。
- 入力行の表示色は常に白（`#ffffff`）。

### 4.11 関数 (`.ezs`)
```
関数名.ezs { ... }
```
- 再利用ブロック（引数なし）。
- 定義は **ソース内の任意位置** に配置可能。
- `}` で閉じる（単独行）。

```
ezs [関数名]
```
- 関数呼び出し。

---

## 5. ブロック構文の詳細

### 5.1 入れ子
- `if` / `loop` は **入れ子可能**。
- `{` を含む行でネストがカウントされます。
- `}` 行でネストがデクリメントされます。

### 5.2 条件式評価
条件式内の `$var` は以下のルールで置換されます。
- 数値ならそのまま。
- 文字列なら `"文字列"` として埋め込み。
- 未定義は `""`。

その後、JavaScript `eval()` によって真偽判定されます。  
評価失敗時は `false`。

---

## 6. 実行モデル

1. 全行を読み込み、空行・コメント行を除外。
2. `.ezs` 定義を先に収集（関数テーブルへ格納）。
3. 定義行以外を先頭から順に実行。

- 変数スコープは **グローバルのみ**。
- `ezs` 内で `set` した変数は全体に影響します。

---

## 7. エラーと未定義動作
- 未知の命令は **無視されます**（エラー表示なし）。
- `calc` / `if` / `loop` の `eval()` 失敗は `0` または `false` として扱われます。
- 文字列内の `+` をそのまま表現する手段はありません。
- 文字列に `" "` を含める方法はありません。

---

## 8. セキュリティ注意
- `calc` と条件式は JavaScript `eval()` を使用します。  
  **外部入力を直接評価しないこと。**

---

## 9. サンプル

### 9.1 カウンタ
```
set i 1
loop $i <= 3 {
  echo "count:" + $i
  set i calc($i + 1)
}
```

### 9.2 入力
```
input name "your name?"
echo "hello " + $name
```
